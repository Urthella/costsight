# Example artifacts

Two kinds of files live here: **input CUR samples** you can upload into the
dashboard, and **committed pipeline outputs** so the repo can be browsed
without first running anything.

## AWS CUR samples — drop into the dashboard's "Upload AWS CUR" button

Real-AWS-CUR-shaped CSVs generated by `python scripts/make_sample_curs.py`.
Each is 60-90 days, long enough for the detectors to flag anomalies (a few
days is not — the models need ~2 weeks of history first).

| File | Scenario | Span | Why grab it |
|---|---|---|---|
| `cur_showcase_90d.csv` | **showcase** | 90 days | **The "see everything" upload** — sustained level shifts (HIGH-severity alerts), spikes and drift across services, so every view fills with strong signal |
| `aws_cur_sample.csv` | spike storm | 90 days | The headline spike demo |
| `cur_default_90d.csv` | default | 90 days | Matches the committed benchmark |
| `cur_spike_storm_60d.csv` | spike storm | 60 days | Many sharp one-day spikes |
| `cur_stealth_leak_90d.csv` | stealth leak | 90 days | Slow hidden multi-service drift |
| `cur_multi_region_90d.csv` | multi-region | 90 days | Costs shift between regions |
| `cur_calm_60d.csv` | calm | 60 days | Almost no anomalies — tests the empty case |
| `cur_loader_fixture.csv` | — | 4 days | Tiny hand-crafted file used by the loader unit test |

## Pipeline outputs — sample results from `python scripts/run_pipeline.py`

Real outputs from one run on the default 90-day synthetic dataset (`seed=42`).

| File | What it is |
|---|---|
| `comparison.csv` | Per-anomaly-type Precision / Recall / F1 for all detectors |
| `alert_quality.csv` | True-positive rate by severity band (LOW / MEDIUM / HIGH) |
| `alerts_stl_sample.json` | Severity-banded alert log produced by the STL detector |
| `attribution_stl_sample.csv` | Root-cause attribution for the STL alerts |
| `benchmark_summary.csv` | 25-seed mean ± std F1 (from `scripts/run_benchmark.py`) |

To regenerate everything: `python scripts/run_pipeline.py` (outputs land under
`outputs/`, which is gitignored) and `python scripts/make_sample_curs.py` for
the CUR samples above.
