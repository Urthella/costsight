# Example artifacts

These files are real outputs from one run of `python scripts/run_pipeline.py`
on the default 90-day synthetic dataset (`seed=42`). They are committed so
the repo can be browsed without first running the pipeline.

| File | What it is |
|---|---|
| `comparison.csv` | Per-anomaly-type Precision / Recall / F1 for all three detectors |
| `alerts_stl_sample.json` | Severity-banded alert log produced by the STL detector |

## AWS CUR samples (drop into the dashboard's "Upload AWS CUR" button)

Real-AWS-CUR-shaped CSVs generated by `python scripts/make_sample_curs.py`.
Each is 60-90 days, long enough for the detectors to flag anomalies (a few
days is not - the models need ~2 weeks of history first).

| File | Scenario | Span |
|---|---|---|
| `aws_cur_sample.csv` | spike storm (the headline demo upload) | 90 days |
| `cur_default_90d.csv` | default mix (matches the committed benchmark) | 90 days |
| `cur_spike_storm_60d.csv` | spike storm | 60 days |
| `cur_stealth_leak_90d.csv` | slow stealth leak | 90 days |
| `cur_multi_region_90d.csv` | multi-region | 90 days |
| `cur_calm_60d.csv` | calm (almost no anomalies - tests the empty case) | 60 days |
| `cur_loader_fixture.csv` | tiny hand-crafted 4-day file used by the loader unit test | 4 days |

To regenerate everything:

```bash
python scripts/run_pipeline.py
```

Outputs land under `outputs/` (which is gitignored).
